particles-free-oscill-euler

Частицы осциллируют относительно центра пространства (как будто к каждой приделана пружина с линейной зависимостью силы от растяжения).
За переключение режимов euler и midpoint отвечает переменная euler
За жесткость пружины отвечает переменная spring
В левом верхнем углу указана характерная энергия системы. Чем меньше она меняется, тем более точен способ симуляции.
Euler работает с меньшим количеством операций, однако энергия системы сохраняется хуже (гарантирует первые 1-2 разряда суммарной энергии).
Midpoint требует большей мощности, зато энергия близка к постоянной (гарантирует первые 3-4 разряда суммарной энергии).

particles-free-oscill-euler-interface

Техническая сторона предыдущего скрипта сохранена, однако с помощью нейросети добавлен удобный интерфейс для подробного сравнения работы методов.
Жесткость и режим работы можно менять прямо во время выполнения с помощью интерфейса.
Можно быстро увеличивать суммарную энергию системы, резко повышая и понижая жесткость пружины (не баг, а фича!).
При испытаниях выяснилось, что при методе Эйлера существует критическое значение энергии (варьируется от запуска), при котором энергия выше этого значения начинает спадать до него
(притом midpoint работает стабильно). Для 100 частиц это можно пронаблюдать, повысив энергию при spring = 0.1 до ~800000 (характерное падения составит 200000).
